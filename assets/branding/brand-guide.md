# WebMCP for WordPress — Brand Guide

## Overview

WebMCP for WordPress is a developer-focused plugin that bridges WordPress capabilities to the WebMCP browser standard. Our brand reflects technical sophistication, trustworthiness, and approachability — positioned alongside companies like Stripe, Vercel, and GitHub.

**Core Message:** "AI agents, meet WordPress." — Structured tools replace scraping and guesswork.

---

## Visual Identity

### Color Palette

**Primary Colors:**
- **WordPress Blue:** `#21759B` — Trustworthy, established, WordPress heritage
  - Used for: Main text, primary UI elements, branding accents
  - Applications: Headlines, icons, borders, gradients

- **Chrome Blue (Google Blue):** `#4285F4` — Modern, connected, cross-platform
  - Used for: Secondary accents, active states, highlight elements
  - Applications: Hover states, secondary headlines, connection visuals, UI polish

**Neutral Colors:**
- **Dark Charcoal:** `#1a1a1a` — Primary text on light backgrounds
- **Soft Background:** `#f8f9fa` — Light gray, professional backgrounds
- **Code Background:** `#1e1e1e` — Dark terminal/editor backgrounds
- **Text Secondary:** `#666666` (light mode), `#adbac7` (dark mode)
- **Borders:** `#e0e0e0` (light), `#3d3d3d` (dark)

**Syntax Highlighting (for code samples):**
- Keywords: `#569cd6` (blue)
- Strings: `#ce9178` (orange)
- Properties: `#9cdcfe` (light blue)
- Functions: `#dcdcaa` (yellow)
- Punctuation: `#d4d4d4` (light gray)

'Cantarell', 'Helvetica Neue', sans-serif

**Weight Usage:**

- **600 Medium:** Section headings, labels, feature titles
- **400 Regular:** Body text, descriptions, paragraphs

**Size Hierarchy:**
- **64px:** Plugin name (main headlines)
- **48px:** Page titles
- **32px:** Section headings
- **28px:** Taglines
- **24px:** Subheadings
- **16px:** Body text, descriptions
- **14px:** Captions, metadata, feature bullets
- **12px:** UI labels, footnotes
- **11px:** Code samples, terminal text

**Line Height:**
- Headlines: 1.2
- Body: 1.6
- Code: 1.4

### Logo & Icon System

**Primary Icon:** Hexagon + WordPress "W" + Circuit Nodes
- Geometric, clean, technical
- Works at all sizes (32px minimum for legibility)
- Center hexagon contains WordPress "W"
- Surrounding nodes suggest API connectivity
- Gradient from WordPress blue to Chrome blue

**Logo Usage:**
- **Full Logo:** Plugin name + icon (horizontal layout) — for large contexts
- **Icon Only:** For favicons, app icons, social media profiles
- **Word Mark Only:** For contexts where color palette must be monochromatic

**No Variations:**
- Never rotate, skew, or distort the logo
- Never add drop shadows or 3D effects
- Never outline or hollow the icon
- Always maintain clear space around the icon (minimum 16px on all sides)

## Visual Language

### Design Principles

1. **Minimalist & Technical**
   - Geometric shapes (hexagons, circles, lines)
   - Clean, flat design — no shadows, gradients used sparingly
   - High contrast text and elements
   - Negative space respected

2. **Code-First Storytelling**
   - Showcase JSON, API responses, and structured data
   - Browser/IDE window mockups with authentic UI chrome
   - Syntax highlighting as a visual design element
   - Terminal/monospace fonts for authenticity

3. **Modern Developer Aesthetic**
   - Dark themes showcase clean, bright text
   - Light themes use pure whites and minimal grays
   - Resembles Stripe, Vercel, GitHub — professional SaaS tools
   - No cartoonish elements, illustrations, or mascots

4. **Accessibility & Clarity**
   - 4.5:1 minimum contrast ratio (WCAG AA)
   - Large, readable text (16px+ for body)
   - Clear visual hierarchy
   - Works in both light and dark modes

### Imagery & Illustrations

- **Use code snippets and structured data as visual elements** — JSON, API responses
- **Browser windows and IDE mockups** — with authentic chrome (traffic lights, URL bars)
- **Geometric shapes and icons** — circles, hexagons, lines, arrows
- **Connection visuals** — nodes linked by traces, representing integration
- **Do NOT use:** clipart, cute mascots, hand-drawn illustrations, photographic backgrounds

### Layout & Spacing

**Grid System:** 8px base unit

**Spacing Scale:**
```
8px, 16px, 24px, 32px, 48px, 64px, 80px
```

**Padding & Margins:**
- Large containers: 60px–80px padding
- Cards/sections: 32px–48px padding
- Element gaps: 12px–24px
- Text spacing: 8px–16px

**Breathing Room:**
- Headlines need 24px+ below
- Paragraphs need 16px gap between
- Feature lists: 12px vertical gap between items
- Code blocks: 24px margin above and below

## Brand Tone

### Voice

- **Professional:** Credible, technical, trustworthy
- **Developer-Centric:** Speak to engineers, value clarity over marketing speak
- **Approachable:** Technical but not elitist — explain concepts clearly
- **Direct:** Say what the plugin does, no corporate jargon
- **Helpful:** Focus on solving problems (no scraping, no guessing)

### Writing Examples

**Good:**
- "Structured tools for AI agents. No scraping. No guesswork."
- "AI agents discover WordPress capabilities as clean JSON tool calls."
- "WebMCP bridges WordPress to the browser standard for AI integration."

**Avoid:**
- "Unlock the power of WordPress AI integration!" (corporate/vague)
- "Your WordPress site just got super smart!" (playful/unprofessional)
- "Revolutionary AI-powered WordPress ecosystem" (overselling)
- Emojis, exclamation marks (except in rare CTAs)

## Color Modes

### Light Mode (Default)
- Background: White or soft gray (`#f8f9fa`)
- Text: Dark charcoal (`#1a1a1a`)
- Accents: WordPress blue (`#21759B`) + Chrome blue (`#4285F4`)
- Code background: Off-white (`#f5f5f5`) or light gray

### Dark Mode
- Background: Near-black (`#0d1117`) or very dark gray (`#1a1a1a`)
- Text: Off-white (`#e0e0e0`) or platform standard (`#adbac7` for GitHub)
- Accents: Brighter WordPress blue (`#4a99d4`) + Chrome blue (unchanged)
- Code background: Darker (`#1e1e1e` or `#010409`)

**Contrast Check:** Verify 4.5:1 WCAG AA compliance for all text/background pairs.

## Dos and Don'ts

### Do:
✓ Use the WordPress + Chrome blue combination to show integration
✓ Showcase code and JSON as visual storytelling
✓ Keep layouts minimal and breathing room generous
✓ Use system fonts for speed and consistency
✓ Apply gradients sparingly (text headings, icon accents)
✓ High contrast everything (dark text on light, vice versa)
✓ Match the aesthetic of Stripe, Vercel, GitHub

### Don't:
✗ Add shadows, 3D effects, or depth
✗ Use decorative or custom fonts
✗ Include clipart, mascots, or cute illustrations
✗ Over-use gradients or transparency
✗ Use more than 2–3 colors per design
✗ Stretch, rotate, or distort the logo
✗ Mix light and dark backgrounds without strong contrast
✗ Use emojis or playful elements in professional contexts

## Future Expansion

As WebMCP for WordPress grows, maintain these principles:

- **Animation:** Smooth, subtle (150–300ms duration, ease-in-out)
- **Illustrations:** If needed, geometric/technical only — vector-based
- **Patterns:** Grid, nodes, hexagons — consistent with icon system
- **Photography:** Avoid; use code/UI mockups instead
- **Accessibility:** Always test color contrast, alt text, keyboard navigation

## Questions & Customization

For questions about applying this brand:

1. **"Can I use a photo of our WordPress site?"** — No. Use a mockup of the code/API interface instead.
2. **"Can I add a tagline under the logo?"** — Yes, if it fits the layout. Keep it under 10 words.
3. **"Can I make the icon look more modern?"** — It's already modern. Test at small sizes before changing.
4. **"Can I use a different blue?"** — No. These blues are carefully chosen to link WordPress and Google/Chrome.
